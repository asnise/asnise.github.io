Skill Tree JSON Documentation

Minimal reference for how to structure skill tree data as JSON for the editor, the game runtime, and external documentation.

1. OverviewEditor format

The Skill Tree Editor exports JSON so that you can treat your entire tree as data. This document focuses on a practical schema that works well for:

  • Visual editing in the browser
  • Game runtime loading and unlock logic
  • Documentation generation (wiki pages, PDFs, etc.)

2. Core concepts

These concepts should be shared between the editor and your game code:

  • Node – a single skill or passive modifier that can be unlocked.
  • Link – a connection between two nodes that defines unlock paths.
  • Stats – attributes changed by the node (for example str, vit, int, dex, luk).
  • Role – design categories such as root, core, keystone.
Keep each tree in a single JSON file and commit it to your repository. The editor can export to this file and the game can read from it without conversion.

3. Recommended JSON schema

The following structure balances editor needs (layout, connections) with game needs (costs, stats, tags). You can rename fields but try to keep the same ideas. 

{
  "version": "1.0.0",
  "meta": {
    "name": "Sacred Geometry Skill Tree",
    "description": "Example configuration for the Skill Tree Editor",
    "createdAt": "2025-11-28T00:00:00Z",
    "author": "Your Name"
  },
  "statsSchema": ["str", "vit", "int", "dex", "luk"],
  "nodes": [
    {
      "id": "n-001",
      "label": "Power Strike",
      "type": "active",
      "role": "core",
      "position": { "x": 0, "y": 0 },
      "radius": 24,
      "parents": [],
      "children": ["n-002"],
      "cost": {
        "points": 1,
        "requiredNodes": []
      },
      "stats": {
        "str": 10,
        "vit": 0,
        "int": 0,
        "dex": 5,
        "luk": 0
      },
      "tags": ["warrior", "melee"],
      "extra": {
        "summary": "Increase physical damage.",
        "description": "Increase physical damage by 15% for melee attacks.",
        "icon": "icons/power-strike.png"
      }
    }
  ],
  "links": [
    {
      "id": "e-001",
      "from": "n-001",
      "to": "n-002",
      "kind": "normal"
    }
  ]
}

Field notes

  • statsSchema defines which stats exist and how to interpret the stats object.
  • position is mainly for the visual layout inside the editor.
  • cost.points is interpreted by your game logic when validating unlocks.
  • extra is free-form for descriptions, icons, and UI-only fields.

4. Documentation JSON

A second JSON file can be dedicated to human-readable documentation. It references the same node ids but groups them into paths or classes. 

{
  "treeName": "Universal Skill Tree",
  "paths": [
    {
      "id": "path-strength",
      "name": "Path of Strength",
      "summary": "Melee and physical damage oriented skills.",
      "nodes": ["n-001", "n-002", "n-010"]
    }
  ],
  "nodeDetails": {
    "n-001": {
      "name": "Power Strike",
      "type": "active",
      "tier": 1,
      "summary": "Increase physical damage.",
      "description": "Increase physical damage by 15% for melee attacks.",
      "requirements": ["None"],
      "effects": ["+10 STR", "+5 DEX"],
      "tags": ["melee", "starter"]
    }
  }
}

5. HTML JSON

For static sites, you can store pre-rendered HTML inside JSON and inject it into the page. This is useful if you generate docs from a build script. 

{
  "sections": [
    {
      "id": "sec-strength",
      "title": "Path of Strength",
      "html": <h2>Path of Strength</h2><p>Melee-focused skills...</p>
    }
  ]
}

6. Typical workflow

  1. Design the tree in the visual editor and export the JSON.
  2. Commit the JSON to your repository.
  3. Load the JSON in your game at startup and build your skill graph and UI.
  4. Optionally, derive documentation JSON or HTML JSON using build scripts.

1. ภาพรวมรูปแบบสำหรับ editor

Skill Tree Editor สามารถ export ข้อมูลเป็น JSON เพื่อให้ทั้งต้นไม้กลายเป็น data ส่วนนี้อธิบายโครงสร้างที่ใช้งานได้จริง เหมาะกับ:

  • ใช้แก้ไขแบบ Visual ในเบราว์เซอร์
  • โหลดในเกมจริงเพื่อใช้ตรวจสอบการปลดล็อก
  • สร้างเอกสารประกอบ เช่น Wiki หรือ PDF

2. แนวคิดหลัก

ตัว editor และเกมควรเข้าใจแนวคิดร่วมกันอย่างน้อยดังนี้:

  • Node – สกิลหรือ passive หนึ่งช่องที่สามารถปลดล็อกได้
  • Link – เส้นเชื่อมระหว่าง node ที่กำหนดเส้นทางการปลดล็อก
  • Stats – ค่าสเตตัสที่ถูกปรับ เช่น str, vit, int, dex, luk
  • Role – บทบาทของ node เช่น root, core, keystone
แนะนำให้ใช้หนึ่งไฟล์ JSON ต่อหนึ่งต้นไม้ แล้ว commit ไว้ใน repository editor สามารถ export ไปยังไฟล์นี้ และเกมก็อ่านใช้ได้เลยโดยไม่ต้องแปลง

3. โครงสร้าง JSON ที่แนะนำ

โครงสร้างด้านล่างพยายามให้ตอบโจทย์ทั้งด้าน layout ใน editor และ logic ในเกม สามารถเปลี่ยนชื่อ field ได้ แต่ควรรักษาแนวคิดเดิมไว้ 

{
  "version": "1.0.0",
  "meta": {
    "name": "Sacred Geometry Skill Tree",
    "description": "ตัวอย่างสกิลทรีสำหรับ Skill Tree Editor",
    "createdAt": "2025-11-28T00:00:00Z",
    "author": "Your Name"
  },
  "statsSchema": ["str", "vit", "int", "dex", "luk"],
  "nodes": [
    {
      "id": "n-001",
      "label": "Power Strike",
      "type": "active",
      "role": "core",
      "position": { "x": 0, "y": 0 },
      "radius": 24,
      "parents": [],
      "children": ["n-002"],
      "cost": {
        "points": 1,
        "requiredNodes": []
      },
      "stats": {
        "str": 10,
        "vit": 0,
        "int": 0,
        "dex": 5,
        "luk": 0
      },
      "tags": ["warrior", "melee"],
      "extra": {
        "summary": "เพิ่มความเสียหายกายภาพ",
        "description": "เพิ่มความเสียหายกายภาพ 15% สำหรับการโจมตีระยะใกล้",
        "icon": "icons/power-strike.png"
      }
    }
  ],
  "links": [
    {
      "id": "e-001",
      "from": "n-001",
      "to": "n-002",
      "kind": "normal"
    }
  ]
}

คำอธิบาย field

  • statsSchema ระบุว่าต้นไม้นี้มีค่าสเตตัสอะไรบ้าง และใช้ตีความ stats อย่างไร
  • position ใช้สำหรับ layout ใน editor เป็นหลัก
  • cost.points ให้เกมเป็นผู้ตีความตอนตรวจสอบการปลดล็อก
  • extra เปิดกว้างสำหรับข้อความอธิบาย ไอคอน และข้อมูลใช้ใน UI เท่านั้น

4. JSON สำหรับเอกสาร

สามารถมีไฟล์ JSON แยกสำหรับเอกสาร ที่จัดกลุ่ม node ตามสายหรือคลาส และอธิบายรายละเอียดในรูปแบบที่อ่านง่าย 

{
  "treeName": "Universal Skill Tree",
  "paths": [
    {
      "id": "path-strength",
      "name": "สายพละกำลัง",
      "summary": "เน้นสกิลโจมตีกายภาพและการต่อสู้ระยะใกล้",
      "nodes": ["n-001", "n-002", "n-010"]
    }
  ],
  "nodeDetails": {
    "n-001": {
      "name": "Power Strike",
      "type": "active",
      "tier": 1,
      "summary": "เพิ่มความเสียหายกายภาพ",
      "description": "เพิ่มความเสียหายกายภาพ 15% สำหรับการโจมตีระยะใกล้",
      "requirements": ["ไม่ต้องการเงื่อนไข"],
      "effects": ["+10 STR", "+5 DEX"],
      "tags": ["melee", "starter"]
    }
  }
}

5. JSON สำหรับ HTML

ในเว็บ static บางกรณี อาจสะดวกกว่าหากสร้าง HTML ล่วงหน้าแล้วเก็บเป็น JSON จากนั้นให้หน้าเว็บนำค่าไปกำหนดเป็น innerHTML ตาม section ที่ต้องการ 

{
  "sections": [
    {
      "id": "sec-strength",
      "title": "สายพละกำลัง",
      "html": <h2>สายพละกำลัง</h2><p>สายสกิลสำหรับผู้เล่นที่เน้นการโจมตีกายภาพระยะใกล้...</p>
    }
  ]
}

6. ลำดับการใช้งานที่แนะนำ

  1. ออกแบบสกิลทรีใน editor แล้ว export เป็นไฟล์ JSON
  2. commit ไฟล์ JSON ไว้ใน repository
  3. ให้เกมโหลด JSON ตอนเริ่มแล้วสร้างกราฟสกิลและ UI จากข้อมูลนั้น
  4. หากต้องการเอกสารเพิ่มเติม ให้สร้าง JSON สำหรับเอกสาร หรือ HTML JSON จาก build script
